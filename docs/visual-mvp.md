# TeleSoccer - Visual MVP

## 1. Objetivo

Definir a camada visual mínima do TeleSoccer antes da Fase 3.

---

## 2. Princípio

O visual MVP não precisa ser sofisticado, mas precisa cumprir duas funções:

1. reforçar a identidade do jogo
2. melhorar a leitura do estado da partida e da carreira

---

## 3. Elementos visuais mínimos

### 3.1 Partida
O usuário deve receber elementos visuais mínimos para:

- placar
- minuto e tempo de jogo
- contexto do lance
- ação escolhida
- gol
- cartão
- lesão
- fim de partida

### 3.2 Carreira
O usuário deve receber elementos visuais mínimos para:

- ficha do jogador
- progresso do atleta
- status da carreira
- entrada no profissional

### 3.3 Multiplayer
O usuário deve conseguir entender visualmente:

- se está em uma sessão compartilhada
- quem é o outro participante
- qual é o estado atual da partida

---

## 4. Formato permitido no MVP

São aceitos formatos simples, como:

- imagens estáticas renderizadas
- cards visuais
- templates com texto + arte
- HUD simples

Não é necessário agora:

- animação avançada
- sprites animados complexos
- engine gráfica completa

---

## 5. Requisitos mínimos de consistência

- identidade visual coerente
- mesma linguagem visual entre menus e partida
- leitura fácil no Telegram
- compatibilidade com futuras expansões

---

## 6. Critérios de aceite

O visual MVP só está pronto quando:

1. o usuário perceber visualmente que está jogando algo e não apenas navegando texto
2. os principais eventos da partida tiverem feedback visual
3. a carreira tiver apresentação visual mínima
4. a camada visual for consistente com o universo do TeleSoccer
